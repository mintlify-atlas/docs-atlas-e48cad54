The Operator configuration file lives at `~/.operator/config.json` (or `$OPERATOR_HOME/config.json` if the environment variable is set). It is loaded once at startup; most fields can also be overridden at runtime via [environment variables](/reference/environment-variables).

  "agents": {
    "defaults": {
      "workspace": "~/.operator/workspace",

      "model_name": "gpt4",
      "max_tokens": 8192,
      "temperature": 0.7,
      "max_tool_iterations": 20
    }
  },
  "model_list": [
    {
      "model_name": "gpt4",
      "model": "openai/gpt-5.2",
      "api_key": "sk-your-openai-key",
      "api_base": "https://api.openai.com/v1"

    {
      "model_name": "claude-sonnet-4.6",

## `agents`

Global agent settings.

<ParamField body="agents.defaults" type="object">
  Default parameters applied to every agent unless overridden at the agent level.

  <Expandable title="properties">
    <ParamField body="workspace" type="string" default="~/.operator/workspace">
      Absolute or `~`-prefixed path to the agent's working directory. The agent reads and writes files here by default. Set `OPERATOR_HOME` to relocate the base directory; the workspace path is then `$OPERATOR_HOME/workspace`.
    </ParamField>

    <ParamField body="restrict_to_workspace" type="boolean" default="true">
      When `true`, the agent cannot read or write files outside the `workspace` directory. Set to `false` to allow full filesystem access (use with care).
    </ParamField>

    <ParamField body="allow_read_outside_workspace" type="boolean" default="false">
      When `true`, the agent can read (but not write) files outside the workspace even if `restrict_to_workspace` is `true`. Useful for referencing read-only reference material.
    </ParamField>

    <ParamField body="model_name" type="string">
      The `model_name` alias to use by default. Must match a `model_name` entry in `model_list`. Example: `"gpt4"`.

      <Note>Prefer `model_name` over the deprecated `model` field. Both are supported during the transition period.</Note>
    </ParamField>

    <ParamField body="max_tokens" type="number" default="32768">
      Maximum number of tokens the model may generate in a single response.
    </ParamField>

    <ParamField body="temperature" type="number" default="null">
      Sampling temperature (0.0–2.0). `null` means the provider default is used. Lower values produce more deterministic output; higher values increase creativity.
    </ParamField>

    <ParamField body="max_tool_iterations" type="number" default="50">
      Maximum number of tool-call/response cycles per agent turn. Prevents runaway loops. Set to `0` for no limit (not recommended).
    </ParamField>

    <ParamField body="image_model" type="string">
      Optional model alias to use for image-generation tasks. Falls back to `model_name` if not set.
    </ParamField>

    <ParamField body="max_media_size" type="number" default="20971520">
      Maximum size in bytes for media attachments (images, files) passed to the model. Default is 20 MB (20 × 1024 × 1024).
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="agents.list" type="object[]">
  Optional list of named agents with per-agent overrides. Each entry can override the model, workspace, and skills for a specific agent.

  <Expandable title="properties">
    <ParamField body="id" type="string" required>
      Unique identifier for the agent. Referenced in `bindings` and `--agent` CLI flags.
    </ParamField>

    <ParamField body="name" type="string">
      Human-readable display name.
    </ParamField>

    <ParamField body="default" type="boolean" default="false">
      If `true`, this agent handles requests that don't match any binding.
    </ParamField>

    <ParamField body="workspace" type="string">
      Per-agent workspace directory. Overrides `agents.defaults.workspace`.
    </ParamField>

    <ParamField body="model" type="string | object">
      Per-agent model. Either a plain string alias (`"gpt4"`) or an object with `primary` and `fallbacks` fields.

      <Expandable title="object form">
        <ParamField body="primary" type="string">
          Primary model alias.
        </ParamField>
        <ParamField body="fallbacks" type="string[]">
          Ordered list of fallback model aliases tried when the primary fails.
        </ParamField>
      </Expandable>
    </ParamField>

    <ParamField body="skills" type="string[]">
      List of skill identifiers pre-loaded for this agent.
    </ParamField>

    <ParamField body="subagents" type="object">
      Configuration for sub-agent delegation.

      <Expandable title="properties">
        <ParamField body="allow_agents" type="string[]">
          Agent IDs this agent may spawn as sub-agents.
        </ParamField>
        <ParamField body="model" type="string | object">
          Model used for sub-agent tasks.
        </ParamField>
      </Expandable>
    </ParamField>
  </Expandable>
</ParamField>

## `model_list`

<Note>
  `model_list` is the recommended way to configure AI providers. The older `providers` top-level key is deprecated and will be removed in v2.0. If only `providers` is present, it is automatically migrated at startup.
</Note>

An ordered array of model entries. Each entry defines an alias, a provider/model identifier, and credentials. Multiple entries with the **same `model_name`** enable automatic round-robin load balancing across them.

```json
{
  "model_list": [
    {
      "model_name": "gpt4",
      "model": "openai/gpt-5.2",
      "api_key": "sk-your-key",
      "api_base": "https://api.openai.com/v1"
    }
  ]
}
```

<ParamField body="model_list[].model_name" type="string" required>
  User-facing alias for this model entry. This is the value you set in `agents.defaults.model_name`. Must be non-empty. Duplicate values across entries are allowed and enable load balancing.
</ParamField>

<ParamField body="model_list[].model" type="string" required>
  Protocol-prefixed model identifier in the form `protocol/model-id`. The protocol determines which provider backend is used. If no prefix is given, `openai` is assumed.

  See the [AI provider reference](/reference/providers) for the full list of supported protocols and example model IDs.
</ParamField>

<ParamField body="model_list[].api_key" type="string">
  API key for authenticating with the provider. Required for all HTTP-based providers unless `api_base` points to a local unauthenticated server.
</ParamField>

<ParamField body="model_list[].api_base" type="string">
  Override the provider's default API endpoint URL. Use this to point at a self-hosted deployment, proxy, or alternative region. Each provider has a compiled-in default that applies when this field is omitted.
</ParamField>

<ParamField body="model_list[].auth_method" type="string">
  Authentication method. Set to `"oauth"` for providers that use OAuth-based login (e.g., `antigravity`, `github-copilot`). Omit for API-key–based providers.
</ParamField>

<ParamField body="model_list[].proxy" type="string">
  HTTP/HTTPS/SOCKS5 proxy URL to use for requests to this provider. Example: `"http://proxy.corp.example.com:8080"`.
</ParamField>

<ParamField body="model_list[].connect_mode" type="string">
  Connection mode for CLI-based providers (`claude-cli`, `codex-cli`, `github-copilot`). Valid values: `"stdio"`, `"grpc"`. Defaults to `"grpc"` for GitHub Copilot.
</ParamField>

<ParamField body="model_list[].workspace" type="string">
  Workspace directory for CLI-based providers (`claude-cli`, `codex-cli`). Defaults to `"."` if not set.
</ParamField>

<ParamField body="model_list[].rpm" type="number">
  Requests-per-minute cap for this entry. When reached, Operator waits before sending the next request.
</ParamField>

<ParamField body="model_list[].max_tokens_field" type="string">
  Override the JSON field name used to pass `max_tokens` in the API request body. Some providers (e.g., newer OpenAI endpoints) require `"max_completion_tokens"` instead of `"max_tokens"`.
</ParamField>

<ParamField body="model_list[].request_timeout" type="number">
  Per-request HTTP timeout in seconds. Values `<= 0` fall back to the compiled default of 120 s.
</ParamField>

### Load balancing

Duplicate a `model_name` across multiple entries to distribute load via round-robin:

```json
{
  "model_list": [
    { "model_name": "gpt4", "model": "openai/gpt-5.2", "api_key": "sk-key1", "api_base": "https://api1.example.com/v1" },
    { "model_name": "gpt4", "model": "openai/gpt-5.2", "api_key": "sk-key2", "api_base": "https://api2.example.com/v1" },
    { "model_name": "gpt4", "model": "openai/gpt-5.2", "api_key": "sk-key3", "api_base": "https://api3.example.com/v1" }
  ]
}
```

Every call to model `gpt4` cycles through the three endpoints in sequence.

## `channels`

Messaging platform integrations. All channels are disabled by default. Enable only the channels you need.

Every channel config includes the following common fields:

| Field | Type | Default | Description |
|---|---|---|---|
| `enabled` | bool | `false` | Enable this channel. |
| `allow_from` | string[] | `[]` | Allowlist of user/account IDs. Empty means all authenticated users are allowed. Accepts both string and numeric IDs. |
| `reasoning_channel_id` | string | `""` | If set, internal reasoning/thinking output is forwarded to this channel or thread ID instead of the main conversation. |

### `tools.exec`

Controls shell command execution safety.

<ParamField body="tools.exec.enable_deny_patterns" type="boolean" default="true">
  When `true`, a built-in set of dangerous command patterns is blocked before execution. Set to `false` to disable all built-in blocking (not recommended in production).

  Blocked by default: `rm -rf`, `format`, `mkfs`, `dd if=`, `shutdown`, `reboot`, `pkill`, `killall`, `sudo`, `chmod`, `chown`, `curl | sh`, `wget | sh`, `docker run`, `docker exec`, `git push --force`, `eval`, and others.

<ParamField body="tools.exec.custom_deny_patterns" type="string[]" default="[]">

  ```json
  {
    "custom_deny_patterns": ["\\brm\\s+-r\\b", "\\bkillall\\s+python"]
  }
  ```
</ParamField>

<ParamField body="tools.exec.custom_allow_patterns" type="string[]" default="[]">
  Regular expressions for commands that are explicitly allowed, overriding any matching deny patterns.

### `tools.mcp`

Model Context Protocol server integration. MCP extends the agent with tools from external servers.

<ParamField body="tools.mcp.enabled" type="boolean" default="false">
  Global toggle for all MCP server connections. Must be `true` for any server to connect.
</ParamField>

<ParamField body="tools.mcp.servers" type="object">
  Map of server names to server configuration objects. Server names are used only for identification in logs.

  <Expandable title="per-server properties">
    <ParamField body="enabled" type="boolean" required>Enable this specific MCP server.</ParamField>

    <ParamField body="type" type="string">
      Transport type: `"stdio"`, `"sse"`, or `"http"`. Auto-detected when omitted: if `url` is set → `sse`; if `command` is set → `stdio`.
    </ParamField>

    <ParamField body="command" type="string">
      Executable to launch for `stdio` transport (e.g., `"npx"`, `"python"`).
    </ParamField>

    <ParamField body="args" type="string[]">
      Arguments passed to `command` for `stdio` transport.
    </ParamField>

    <ParamField body="env" type="object">
      Environment variables injected into the `stdio` process. Use this for API keys the MCP server needs.
    </ParamField>

    <ParamField body="env_file" type="string">
      Path to a `.env`-format file whose variables are loaded into the `stdio` process environment.
    </ParamField>

    <ParamField body="url" type="string">
      Endpoint URL for `sse` or `http` transport.
    </ParamField>

    <ParamField body="headers" type="object">
      HTTP headers sent with every request to `sse`/`http` servers. Commonly used for `Authorization` tokens.
    </ParamField>
  </Expandable>

  **Stdio example:**

  ```json
  {
    "tools": {
      "mcp": {
        "enabled": true,
        "servers": {
          "filesystem": {
            "enabled": true,
            "command": "npx",
            "args": ["-y", "@modelcontextprotocol/server-filesystem", "/tmp"]
          }
        }
      }
    }
  }
  ```

  **Remote SSE/HTTP example:**

  ```json
  {
    "tools": {
      "mcp": {
        "enabled": true,
        "servers": {
          "context7": {
            "enabled": true,
            "type": "http",
            "url": "https://mcp.context7.com/mcp",
            "headers": { "CONTEXT7_API_KEY": "ctx7sk-xx" }
          }
        }
      }
    }
  }
  ```
</ParamField>

<Warning>
  MCP server configuration (`tools.mcp.servers.*`) cannot be set via environment variables. Use `config.json` for all server definitions.
</Warning>

## Deprecated: `providers`

<Warning>
  The top-level `providers` key is deprecated as of v1.x and will be removed in v2.0. Migrate to `model_list` instead. See the [migration guide](/configuration/models) for details.

  If `model_list` is empty and `providers` is present, Operator performs an automatic in-memory migration at startup.
</Warning>

The legacy format configured providers globally rather than per-model:

```json
{
  "providers": {
    "openai": { "api_key": "sk-...", "api_base": "https://api.openai.com/v1" },
    "anthropic": { "api_key": "sk-ant-..." }
  },
  "agents": {
    "defaults": { "provider": "openai", "model": "gpt-5.2" }
  }
}
```
